<feature>
  <name>calculatePriceEstimate — itemized booking cost calculator</name>
  <files>src/lib/price-estimate.ts, tests/lib/price-estimate.test.ts</files>
  <behavior>
    Input type PriceInput:
      - checkin: string | undefined (YYYY-MM-DD, local date)
      - checkout: string | undefined (YYYY-MM-DD, local date)
      - numGuests: number
      - selectedAddOnIds: string[]
      - addOns: Array of { id: string, name: string, price: number }
      - baseNightlyRate: number
      - cleaningFee: number
      - extraGuestFee: number (per extra guest per night)
      - baseGuests: number (guests included in base rate)
      - serviceFeePercent: number (e.g. 10 for 10%)
      - depositAmount: number

    Return type PriceEstimate | null:
      - null if checkin or checkout is missing or nights <= 0
      - nights: number (differenceInDays checkout - checkin)
      - nightlyTotal: number (nights * baseNightlyRate)
      - cleaningFee: number (pass-through)
      - extraGuestTotal: number (max(0, numGuests - baseGuests) * extraGuestFee * nights)
      - addOnTotal: number (sum of prices for selected add-on IDs only)
      - depositAmount: number (pass-through)
      - serviceFee: number (subtotal * serviceFeePercent / 100)
      - serviceFeePercent: number (pass-through for display)
      - total: number (subtotal + serviceFee)

    Test cases:
      - Returns null when checkin is undefined
      - Returns null when checkout is undefined
      - Returns null when checkout is same day as checkin (nights = 0)
      - 3 nights @ $120/night, 2 guests, baseGuests=2 → nightlyTotal=$360, extraGuestTotal=$0
      - 3 nights @ $120/night, 4 guests, baseGuests=2, extraGuestFee=$10 → extraGuestTotal=$60 (2 extra × $10 × 3 nights)
      - Cleaning fee $50 always included regardless of guests
      - 2 add-ons selected ($20 + $0) → addOnTotal=$20
      - Service fee 10% on subtotal of $420 ($360 + $50 + $10 deposit) → serviceFee=$42, total=$462
      - Deposit $100 always included in subtotal
      - Unselected add-on IDs are NOT included in addOnTotal
  </behavior>
  <implementation>
Create `src/lib/price-estimate.ts` as a pure function file with no imports except `date-fns`.

Use `differenceInDays` from `date-fns` for night calculation to avoid DST bugs.
Parse checkin/checkout strings as `new Date(s + "T00:00:00")` — established Phase 2/3 pattern for UTC off-by-one prevention.

Export: `calculatePriceEstimate`, `PriceInput` type, `PriceEstimate` type.
  </implementation>
</feature>
