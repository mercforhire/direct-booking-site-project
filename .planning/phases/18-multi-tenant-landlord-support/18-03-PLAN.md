# Plan 18-03: Guest Route Migration (File Structure)

## Goal
Move all guest-facing RSC routes from `src/app/` to `src/app/[landlord]/`, make each page accept `params.landlord` and resolve the landlord from DB, update all internal links to include the `/{slug}/` prefix, and scope room/booking queries by landlordId.

## must_haves
- [ ] All guest-facing pages exist under `src/app/[landlord]/`
- [ ] Each moved page accepts `params: Promise<{ landlord: string }>` and resolves the landlord slug
- [ ] Room listing page filters by `landlordId`
- [ ] My-bookings page filters by both `guestUserId` AND landlord's rooms
- [ ] All `<Link href>` values in moved pages include `/${slug}/` prefix
- [ ] Root `src/app/page.tsx` redirects to `/highhill`
- [ ] Old route files are deleted (or replaced with redirects)

## Tasks

<task id="18-03-01">
<title>Create src/app/[landlord]/page.tsx from existing homepage</title>
<read_first>
- `src/app/page.tsx` — current homepage with hardcoded values and links
</read_first>
<action>
1. Create directory: `src/app/[landlord]/`
2. Copy `src/app/page.tsx` to `src/app/[landlord]/page.tsx`
3. Modify the new file:

   a. Change the function signature from:
   ```typescript
   export default async function HomePage() {
   ```
   to:
   ```typescript
   export default async function LandlordHomePage({
     params,
   }: {
     params: Promise<{ landlord: string }>
   }) {
     const { landlord: slug } = await params
   ```

   b. Add import at top: `import { getLandlordBySlug } from "@/lib/landlord"`
   c. Add import: `import { notFound } from "next/navigation"`
   d. After resolving slug, fetch landlord:
   ```typescript
   const landlordData = await getLandlordBySlug(slug)
   if (!landlordData) notFound()
   ```

   e. Change the room query to filter by landlordId:
   ```typescript
   const rooms = await prisma.room.findMany({
     where: { photos: { some: {} }, landlordId: landlordData.id },
     include: { photos: { orderBy: { position: "asc" }, take: 1 } },
   })
   ```

   f. Update ALL internal `<Link href>` values to include slug prefix:
   - `href="/rooms"` -> `href={`/${slug}/rooms`}`
   - `href={`/rooms/${room.id}`}` -> `href={`/${slug}/rooms/${room.id}`}`
   - `href="/guest/login?next=/my-bookings"` -> `href={`/${slug}/guest/login?next=/${slug}/my-bookings`}`
   - `href="/guest/signup"` -> `href={`/${slug}/guest/signup`}`
   - `href="/login"` stays as-is (admin login is not landlord-scoped)

   g. Replace hardcoded text (will be completed more thoroughly in Plan 04, but start here):
   - `Leon&rsquo;s Home` -> `{landlordData.name}`
   - `9 Highhill Dr &middot; Scarborough, ON` -> `{landlordData.address}`
   - `Leon&rsquo;s` in headline -> `{landlordData.ownerName}&rsquo;s`
   - `"Direct contact with Leon"` -> `` `Direct contact with ${landlordData.ownerName}` ``
   - `"Book directly with Leon"` -> `` `Book directly with ${landlordData.ownerName}` ``
</action>
