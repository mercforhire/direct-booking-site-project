**BookingRangePicker** (`src/components/guest/booking-range-picker.tsx`):

Implements Pattern 2 from the research doc exactly. Props:
```typescript
interface BookingRangePickerProps {
  blockedDateStrings: string[]       // ["2025-05-15", ...] — YYYY-MM-DD local dates
  bookingWindowMonths: number
  minStayNights: number
  maxStayNights: number
  value: DateRange | undefined       // from react-day-picker
  onChange: (range: DateRange | undefined) => void
}
```

Implementation:
- Parse each blockedDateString as `new Date(s + "T00:00:00")` — prevents UTC off-by-one
- `today = new Date(); today.setHours(0,0,0,0)`
- `windowEnd` = today + bookingWindowMonths months (use `addMonths` from date-fns)
- `<DayPicker mode="range" selected={value} onSelect={onChange} min={minStayNights} max={maxStayNights} disabled={[{ before: today }, { after: windowEnd }, ...blockedDates]} excludeDisabled modifiers={{ blocked: blockedDates }} modifiersClassNames={{ blocked: "line-through opacity-50" }} />`
- Import `"react-day-picker/style.css"` at top

**BookingForm** (`src/components/guest/booking-form.tsx`):

`"use client"` component. Uses `react-hook-form` with `zodResolver(bookingSchema)`.

```typescript
  numGuests: defaultGuests,

  noteToLandlord: "",
  guestName: "",
  guestEmail: "",
  guestPhone: "",

  password: "",
}
```

Form sections in order (matching locked decision from CONTEXT.md):
1. **Dates + guests**: `<BookingRangePicker>` controlled by RHF `watch("checkin")/watch("checkout")` — use `setValue("checkin", ...)` in `onChange`. Guest count: `<Input type="number" min={1} max={room.maxGuests}>` registered with RHF.
2. **Add-ons**: One `<Checkbox>` per add-on. Label: "[name] — [price === 0 ? 'Free' : '$'+price]". Controlled via RHF `watch("selectedAddOnIds")` + `setValue`.
3. **Note to landlord**: `<textarea>` (or multi-line `<Input>`) registered with RHF. Placeholder: "Any questions or special requests?"
4. **Guest info**: Name, Email, Phone `<Input>` fields with labels and error messages.
5. **Account creation**: Checkbox "Create a free account to view this booking anytime". When checked (watch("createAccount")), show Password `<Input type="password">` inline below.

Live pricing: `useMemo` on all watched values → calls `calculatePriceEstimate(...)` → passes `estimate` to `<BookingPriceSummary>`. Also `setValue("estimatedTotal", estimate?.total ?? 0)` via `useEffect` when estimate changes.

Submit validation: Submit button is `disabled` when:
- `formState.isSubmitting` (double-click guard)
- Selected date range contains a blocked date (check via the blockedDateStrings array)
- Selected dates fall outside booking window
- Night count violates minStayNights or maxStayNights
- No checkin or checkout selected

Desktop layout: `<div className="grid grid-cols-1 md:grid-cols-[1fr_320px] gap-8 items-start">`. Form on left. `<BookingPriceSummary className="md:sticky md:top-8">` on right.
Mobile: BookingPriceSummary renders inside the `<details>` accordion below the form (see Task 1).

On submit: call `submitBooking(data)` (imported from `@/actions/booking` — this import will be wired after plan 04-03 is complete; for now, add a TODO comment if 04-03 is not yet done, but wire it directly since plans 04-03 and 04-04 are in the same wave). Handle `{ error }` return — show a top-of-form error message via `useState`.

Note: `redirect()` in the server action will throw (Next.js behavior) — the form component does not need to handle the success redirect explicitly.
