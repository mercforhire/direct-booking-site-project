    Test cases for submitBooking:
      - Validates input with bookingSchemaCoerced; returns {error} on invalid data
      - With createAccount=false: calls prisma.booking.create with guestUserId=null
      - With createAccount=true and valid password: calls supabase.auth.signUp with email, password, and options.data.role="guest"
      - With createAccount=true and signUp returning user: stores user.id in guestUserId
      - With createAccount=true and signUp returning user=null (duplicate email): stores guestUserId=null (no error thrown)
      - Stores selectedAddOnIds as array (not string)
      - Stores noteToLandlord when provided; null when omitted
      - Generated accessToken is a UUID string (matches /^[0-9a-f-]{36}$/)
      - Calls resend.emails.send with to=guestEmail and subject containing "received"
      - Calls redirect("/bookings/[created.id]?new=1") after successful create

    Note: redirect() throws in Next.js (that's how it works). Tests should use vi.mocked(redirect).mock.calls to verify it was called, OR wrap the action call in a try/catch and verify the redirect was attempted.
  </behavior>
