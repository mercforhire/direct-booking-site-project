# Plan 17-02: Auth-Aware Booking Form

## Goal
When a logged-in guest visits `/rooms/[id]/book`, their name/email/phone are prefilled and read-only, Section 5 ("Save my booking") is replaced with a signed-in banner, and their booking is automatically linked to their account. The nav adapts based on auth state.

## Scope
- `src/app/rooms/[id]/book/page.tsx` — add `getUser()`, pass prefill props, update nav
- `src/components/guest/booking-form.tsx` — new props, conditional render for logged-in state
- `src/lib/validations/booking.ts` — add optional `guestUserId` field
- `src/actions/booking.ts` — use `guestUserId` when provided (skip account creation)

---

### Task 3 — Update booking form RSC to call `getUser()`

**File:** `src/app/rooms/[id]/book/page.tsx`

Add import:
```ts
import { createClient } from "@/lib/supabase/server"
```

In the RSC body (after the `settings` query, before the `return`):
```ts
const supabase = await createClient()
const { data: { user } } = await supabase.auth.getUser()

const isLoggedIn = !!user
const guestUserId = user?.id ?? undefined
const prefillData = isLoggedIn
  ? {
      name: (user!.user_metadata?.name as string) ?? "",
      email: user!.email ?? "",
      phone: (user!.user_metadata?.phone as string) ?? "",
    }
  : undefined
```

Update the `<BookingForm>` call to pass new props:
```tsx
<BookingForm
  room={{ ... }}          // unchanged
  settings={{ ... }}      // unchanged
  blockedDateStrings={blockedDateStrings}
  perDayRates={perDayRates}
  defaultCheckin={checkin}
  defaultCheckout={checkout}
  defaultGuests={guests ? parseInt(guests, 10) : 1}
  isLoggedIn={isLoggedIn}
  prefillData={prefillData}
  guestUserId={guestUserId}
/>
```

Place this hidden input inside the `<form>` element, just before the submit button.

**Step 4d — Make Section 4 fields read-only when logged in:**

When `isLoggedIn`, add `readOnly` and a dimmed style to the name/email/phone inputs:

```tsx
<Input
  id="guestName"
  type="text"
```

Apply the same `readOnly={isLoggedIn}` + dimmed style to `guestEmail` and `guestPhone` inputs.
