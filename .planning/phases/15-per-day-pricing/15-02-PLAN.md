    1. Add `perDayRates?: Record<string, number>` to PriceInput type (optional field, no breaking change).

    2. Replace `const nightlyTotal = nights * baseNightlyRate` with a loop:
       ```typescript
       const checkinDate = new Date(checkin + "T12:00:00.000Z")
       const checkoutDate = new Date(checkout + "T12:00:00.000Z")
       // nights already computed via differenceInDays — keep that unchanged
       let nightlyTotal = 0
       if (input.perDayRates) {
         const cursor = new Date(checkinDate)
         while (cursor < checkoutDate) {
           const key = cursor.toISOString().slice(0, 10)
           nightlyTotal += input.perDayRates[key] ?? baseNightlyRate
           cursor.setUTCDate(cursor.getUTCDate() + 1)
         }
       } else {
         nightlyTotal = nights * baseNightlyRate
       }
       ```
       Use setUTCDate(getUTCDate() + 1) — DST safety per Phase 14 decision.
       Use toISOString().slice(0, 10) for key derivation — consistent with noon-UTC writes.

    3. All other fields (extraGuestTotal, addOnTotal, serviceFee, total) remain unchanged — they use nightlyTotal as a scalar, which the loop now computes correctly.

    4. Do NOT remove the differenceInDays import or the existing `nights` variable — nights is still needed for extraGuestTotal = Math.max(0, numGuests - baseGuests) * extraGuestFee * nights.
  </implementation>
