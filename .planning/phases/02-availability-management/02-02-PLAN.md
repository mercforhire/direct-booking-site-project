Date normalization rule (from research):
- Server: `b.date.toISOString().slice(0, 10)` → string "YYYY-MM-DD" passed to client
- Client: `new Date(str + "T00:00:00.000Z")` to reconstruct Date for DayPicker
- Server action receives: "YYYY-MM-DD" string, reconstructs with `new Date(str + "T00:00:00.000Z")`
