# Requirements Archive: v1.0 MVP

**Archived:** 2026-04-01
**Original defined:** 2026-03-25
**Core Value:** Repeat guests can book a room directly with the landlord without going through Airbnb, saving both parties on platform fees.

---

## v1 Requirements (All Outcomes Recorded)

### Room Listings

- [x] **ROOM-01**: Guest can browse all rooms with photos and written description — ✓ Validated v1.0 (Phase 3)
- [x] **ROOM-02**: Guest can see the estimated nightly rate on each room listing — ✓ Validated v1.0 (Phase 3, enhanced with per-day rates in Phase 15)
- [x] **ROOM-03**: Guest can see the full fee structure on each room listing — ✓ Validated v1.0 (Phase 3)
- [x] **ROOM-04**: Each room listing shows the maximum guest capacity — ✓ Validated v1.0 (Phase 3)

### Availability

- [x] **AVAIL-01**: Guest can view a room's availability calendar showing which dates are blocked or available — ✓ Validated v1.0 (Phase 2, display correctness fixed in Phase 14)
- [x] **AVAIL-02**: Landlord can manually block and unblock specific dates per room from the admin dashboard — ✓ Validated v1.0 (Phase 2, ET fix in Phase 14)
- [x] **AVAIL-03**: Landlord can configure the global booking window (3–9 months ahead) — ✓ Validated v1.0 (Phase 2)
- [x] **AVAIL-04**: Landlord can set minimum and maximum stay length per room — ✓ Validated v1.0 (Phase 2)

### Booking Request

- [x] **BOOK-01**: Guest can submit a booking request specifying room, check-in/check-out dates, and number of guests — ✓ Validated v1.0 (Phase 4)
- [x] **BOOK-02**: Guest sees a full itemized price estimate before submitting — ✓ Validated v1.0 (Phase 4, per-day rates added Phase 15)
- [x] **BOOK-03**: Guest can select per-room add-on options at request time — ✓ Validated v1.0 (Phase 4)
- [x] **BOOK-04**: Guest can include a note or message to the landlord with their request — ✓ Validated v1.0 (Phase 4)
- [x] **BOOK-05**: Guest can submit a booking request without creating an account — ✓ Validated v1.0 (Phase 4, middleware fix in Phase 10)
- [x] **BOOK-06**: Guest can optionally create an account to view their booking history — ✓ Validated v1.0 (Phase 4 + Phase 17 sign-up flow)

### Approval Flow

- [x] **APPR-01**: Landlord receives an email notification when a new booking request is submitted — ✓ Validated v1.0 (Phase 5)
- [x] **APPR-02**: Landlord can approve a booking request and set the exact confirmed price — ✓ Validated v1.0 (Phase 5)
- [x] **APPR-03**: Landlord can decline a booking request with an optional reason — ✓ Validated v1.0 (Phase 5)
- [x] **APPR-04**: Guest receives an email when their request is approved — ✓ Validated v1.0 (Phase 5, email CTA fixed in Phase 10)
- [x] **APPR-05**: Guest receives an email when their request is declined — ✓ Validated v1.0 (Phase 5, email CTA fixed in Phase 10)

### Payment

- [x] **PAY-01**: Approved guest can pay online via Stripe Checkout — ✓ Validated v1.0 (Phase 6)
- [x] **PAY-02**: Approved guest can pay by e-transfer; landlord manually marks the booking as paid — ✓ Validated v1.0 (Phase 6)
- [x] **PAY-03**: Landlord can configure an adjustable service fee — ✓ Validated v1.0 (Phase 6)
- [x] **PAY-04**: Landlord can configure an optional deposit amount required per booking — ✓ Validated v1.0 (Phase 6)

### Booking Extensions (Deferred)

- [ ] **EXT-01**: Guest can submit a request to extend an existing booking — *Deferred v1.1 (infrastructure built Phase 7; guest req deferred)*
- [ ] **EXT-02**: Landlord receives email notification for extension request — *Deferred v1.1*
- [ ] **EXT-03**: Landlord can approve an extension and set the additional price — *Deferred v1.1*
- [ ] **EXT-04**: Landlord can decline an extension request — *Deferred v1.1*
- [ ] **EXT-05**: Guest receives email notification of extension approval or decline — *Deferred v1.1*
- [ ] **EXT-06**: Guest can pay the extension amount via Stripe or e-transfer — *Deferred v1.1*

### Cancellations & Refunds

- [x] **CNCL-01**: Landlord can cancel any booking from the admin dashboard at any time — ✓ Validated v1.0 (Phase 8)
- [x] **CNCL-02**: When cancelling, landlord enters the refund amount (no fixed policy) — ✓ Validated v1.0 (Phase 8)
- [x] **CNCL-03**: For Stripe-paid bookings, system automatically issues Stripe refund — ✓ Validated v1.0 (Phase 8)
- [x] **CNCL-04**: For e-transfer bookings, landlord manually marks as refunded — ✓ Validated v1.0 (Phase 8)
- [x] **CNCL-05**: If cancelled before check-in, deposit is always included in refundable amount — ✓ Validated v1.0 (Phase 8)
- [x] **CNCL-06**: If cancelled mid-stay, landlord decides whether to include deposit — ✓ Validated v1.0 (Phase 8)
- [x] **CNCL-07**: Guest receives an email on cancellation with refund amount and timeline — ✓ Validated v1.0 (Phase 8)

### Guest Booking Page

- [x] **GUEST-01**: Guest can view a current booking page showing booking details — ✓ Validated v1.0 (Phase 4, middleware fix Phase 10)
- [ ] **GUEST-02**: Guest can view their extension request status from the booking page — *Deferred v1.1*
- [ ] **GUEST-03**: Guest can submit an extension request directly from the booking page — *Deferred v1.1*

### Messaging

- [x] **MSG-01**: Guest can send a text message to the landlord from their booking page — ✓ Validated v1.0 (Phase 9)
- [x] **MSG-02**: Landlord can send a text message to the guest from the admin dashboard — ✓ Validated v1.0 (Phase 9)
- [x] **MSG-03**: Both parties can view the full message thread (booking-scoped) — ✓ Validated v1.0 (Phase 9)
- [x] **MSG-04**: Landlord receives email notification when a guest sends a message — ✓ Validated v1.0 (Phase 9)
- [x] **MSG-05**: Guest receives email notification when the landlord replies — ✓ Validated v1.0 (Phase 9)

### Admin Dashboard

- [x] **ADMIN-01**: Landlord can view all bookings organized by status — ✓ Validated v1.0 (Phase 5)
- [x] **ADMIN-02**: Landlord can add and edit room listings — ✓ Validated v1.0 (Phase 1)
- [x] **ADMIN-03**: Landlord can configure per-room fees — ✓ Validated v1.0 (Phase 1)
- [x] **ADMIN-04**: Landlord can manage room availability — ✓ Validated v1.0 (Phase 2)
- [x] **ADMIN-05**: Landlord can configure global settings — ✓ Validated v1.0 (Phase 1)
