<task id="18-07-04">
<title>Add legacy redirect for /bookings/[id] to preserve existing email links</title>
<read_first>
- (none — creating a new fallback route)
</read_first>
<action>
Existing booking confirmation emails contain links like `uptrendinvestments.net/bookings/{id}?token=...`. After migration, these URLs would 404 because the route moved to `/{slug}/bookings/{id}`.

Create a fallback route at `src/app/bookings/[id]/page.tsx` (or `route.ts`) that looks up the booking's room -> landlord -> slug and redirects:

```typescript
import { redirect, notFound } from "next/navigation"
import { prisma } from "@/lib/prisma"

export default async function LegacyBookingRedirect({
  params,
  searchParams,
}: {
  params: Promise<{ id: string }>
  searchParams: Promise<Record<string, string | undefined>>
}) {
  const { id } = await params
  const sp = await searchParams

  const booking = await prisma.booking.findUnique({
    where: { id },
    select: { room: { select: { landlord: { select: { slug: true } } } } },
  })

  if (!booking) notFound()

  const slug = booking.room.landlord.slug
  const queryString = new URLSearchParams(
    Object.entries(sp).filter((e): e is [string, string] => e[1] !== undefined)
  ).toString()

  redirect(`/${slug}/bookings/${id}${queryString ? `?${queryString}` : ""}`)
}
```

This preserves backward compatibility for existing email links.
</action>
