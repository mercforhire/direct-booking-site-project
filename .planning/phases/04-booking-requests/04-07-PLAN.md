Remove the existing block (lines 24-33):
```typescript
// Mock Supabase server client
const mockSignUp = vi.fn()

```

Replace with a mock that intercepts `@supabase/supabase-js` and returns the admin methods the action actually calls. Use `vi.hoisted` so the mock refs are available in the factory:

```typescript
const { mockAdminCreateUser, mockAdminListUsers } = vi.hoisted(() => ({
  mockAdminCreateUser: vi.fn(),
  mockAdminListUsers: vi.fn(),

      },
    },
  })),
}))
```

**3. Update the redirect assertion.**

Line 200 currently:
```typescript
expect(vi.mocked(redirect)).toHaveBeenCalledWith("/bookings/booking-123?new=1")
```

The accessToken in the test is `mockCreatedBooking.accessToken = "some-uuid"`. However, the action generates its own `crypto.randomUUID()` — the mock booking's accessToken is what prisma.booking.create returns (stored value), not the token used in the redirect (the token generated before the create call).

```typescript
expect(vi.mocked(redirect)).toHaveBeenCalledWith(

)
```
