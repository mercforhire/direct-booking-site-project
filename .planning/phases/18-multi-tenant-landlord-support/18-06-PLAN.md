# Plan 18-06: Email & Deep Link URLs

## Goal
Fix all URLs in server actions that generate deep links (Stripe success/cancel URLs, booking redirect after submit, revalidatePath calls) and replace `process.env.LANDLORD_EMAIL` with the landlord's email from the database, so all email notifications and redirects use the correct `/{slug}/` prefixed paths.

## must_haves
- [ ] `submitBooking` redirects to `/{slug}/bookings/{id}` (not `/bookings/{id}`)
- [ ] Stripe `success_url` and `cancel_url` include `/{slug}/` prefix
- [ ] `process.env.LANDLORD_EMAIL` replaced with `landlord.email` from DB in all 4 action files
- [ ] All `revalidatePath` calls for guest-facing paths include `/{slug}/` prefix
- [ ] Email notification for landlord uses DB email, not env var

## Tasks

<task id="18-06-01">
<title>Update submitBooking redirect and landlord email notification</title>
<read_first>
- `src/actions/booking.ts` — submitBooking with redirect to /bookings/{id} and LANDLORD_EMAIL usage
</read_first>
<action>
In `src/actions/booking.ts`:

1. Add import: `import { prisma } from "@/lib/prisma"` (if not already imported — it already is)
2. The `submitBooking` function needs the landlord slug for the redirect. Since it receives `roomId`, look up the landlord via the room:

   After `const created = await prisma.booking.create(...)`, add:
   ```typescript
   const landlord = await prisma.landlord.findUnique({
     where: { id: created.room.landlordId },
     select: { slug: true, email: true, ownerName: true },
   })
   ```
   Note: The `created` object needs to include `room.landlordId`. Update the `include` in the booking create:
   ```typescript
   include: { room: { select: { name: true, landlordId: true } } },
   ```

3. Replace the landlord email notification block:
   Change:
   ```typescript
   if (process.env.LANDLORD_EMAIL) {
   ```
   to:
   ```typescript
   if (landlord?.email) {
   ```
   And change:
   ```typescript
   to: process.env.LANDLORD_EMAIL,
   ```
   to:
   ```typescript
   to: landlord.email,
   ```

4. Update the redirect at the end:
   Change:
   ```typescript
   redirect(`/bookings/${created.id}?token=${accessToken}&new=1`)
   ```
   to:
   ```typescript
   redirect(`/${landlord?.slug ?? "highhill"}/bookings/${created.id}?token=${accessToken}&new=1`)
   ```
</action>
<acceptance_criteria>
- `grep "LANDLORD_EMAIL" src/actions/booking.ts` returns no results
- `grep "landlord.email" src/actions/booking.ts` returns the email recipient
- `grep "landlord?.slug" src/actions/booking.ts` returns the redirect with slug
- `grep "landlordId" src/actions/booking.ts` returns the room include
</acceptance_criteria>
</task>
