Complete Phase 7 booking extension feature:
- Guest can request an extension from /bookings/[id] (inline DayPicker form)
- Guest can cancel a pending extension request
- Admin sees "Extension pending" badge on /admin/bookings list
- Admin can approve (with price), decline (with optional reason), and mark extension as paid from /admin/bookings/[id]
- Guest sees extension status section (PENDING/APPROVED/DECLINED/PAID) with payment options when APPROVED
- Stripe Checkout and e-transfer payment flows for extension
- Email notifications at each step
- Booking checkout date updates when extension is paid
  </what-built>
  <how-to-verify>
Run `npm run dev` then test these scenarios:

**Scenario 1: Submit extension request**
1. Open a booking at /bookings/[id] for a PAID or APPROVED booking (use token param)
2. Scroll below the payment section
3. Confirm "Extension Request" section is visible
4. Click "Request Extension" button — confirm inline form appears (DayPicker)
5. Select a date after the current checkout date
6. Submit — confirm inline success message appears ("Extension request submitted — we'll email you once it's reviewed.")
7. Confirm button disappears while PENDING

**Scenario 2: Admin sees Extension pending badge**
1. Visit /admin/bookings
2. Confirm the booking with the pending extension shows "Extension pending" badge

**Scenario 3: Admin approves extension**
1. Visit /admin/bookings/[id] for the booking with pending extension
2. Scroll to "Extension Request" section
3. Confirm requested checkout date and any guest note are shown
4. Enter extension price and click "Approve"
5. Confirm AlertDialog appears — click Approve to confirm
6. Page refreshes — confirm extension status shows as Approved

**Scenario 4: Guest sees APPROVED state with payment options**
1. Return to /bookings/[id]
2. Confirm extension section shows APPROVED status, extension price, Stripe button, and e-transfer instructions

**Scenario 5: Decline extension (separate booking)**
1. Create another extension request via Scenario 1
2. In admin, click "Decline" (with optional reason)
3. In guest view, confirm DECLINED state shows decline reason
4. Confirm "Request Extension" button is re-enabled after decline

**Scenario 6: Mark extension as paid (admin)**
1. Approve an extension (Scenario 3)
2. In admin detail, click "Mark Extension as Paid"
3. Confirm booking checkout date updates to the extension's requestedCheckout date
4. Visit /bookings/[id] — confirm extension shows PAID state and main booking dates updated

**Scenario 7: Stripe extension payment (optional if Stripe test mode configured)**
1. With an APPROVED extension, click "Pay by Card" on /bookings/[id]
2. Complete Stripe test checkout
3. Confirm redirect to /bookings/[id]?extension_paid=1 with "Extension payment confirmed" banner
4. Confirm extension status PAID and booking checkout date updated

**Email checks (if RESEND configured):**
- Extension request → landlord receives "Extension request — [room name]"
- Approve → guest receives approval email with price and payment link
- Decline → guest receives decline email with optional reason
