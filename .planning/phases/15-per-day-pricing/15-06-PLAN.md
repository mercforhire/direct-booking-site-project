<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Per-day pricing for the admin availability calendar and guest booking form:
    - Admin calendar shows effective price per available date (override in blue, base rate in gray)
    - Single click on a date opens an edit panel below the calendar with block toggle and price input
    - Empty price field = use base rate; entering a price saves it as an override
    - Range mode has a "Set Range Price" button that applies a price to all selected dates
    - Guest booking form nightly total uses per-day override prices
  </what-built>
  <how-to-verify>
    Start the dev server: `npm run dev`

    Scenario 1 — Price display on calendar:
    1. Visit http://localhost:3000/availability
    2. Select a room
    3. Confirm every available (non-blocked) date shows a price below the day number
    4. Confirm blocked dates show NO price
    5. Confirm base-rate dates show price in muted gray text
    (overrides will be in blue once you add one in Scenario 2)

    Scenario 2 — Single date price override:
    1. Click on any available date
    2. Confirm an edit panel appears below the calendar showing that date
    3. Enter a price (e.g., 250) and press Enter or click Done
    4. Confirm the calendar tile for that date now shows $250 in blue bold text
    5. Calendar should update without a full page reload (optimistic)

    Scenario 3 — Clear a price override:
    1. Click the same date from Scenario 2
    2. Clear the price input field (empty it completely)
    3. Press Enter or click Done
    4. Confirm the tile reverts to the base rate in gray

    Scenario 4 — Range price:
    1. Click "Select Range" and select a 3-day range
    2. Confirm "Set Range Price" button appears alongside Block Range / Unblock Range
    3. Click "Set Range Price", enter 180, click Apply
    4. Confirm all 3 dates in the range now show $180 in blue

    Scenario 5 — Block toggle in edit panel:
    1. Click an available date
    2. Check the "Blocked" checkbox in the edit panel
    3. Click Done
    4. Confirm the date is now shown as blocked (red background) with no price badge

    Scenario 6 — Guest booking form with override:
    1. Visit http://localhost:3000/rooms (pick any room)
    2. Click "Book" or navigate to the booking page
    3. Select a date range that includes the date from Scenario 2 (the $250 override)
    4. Confirm the "Nightly total" in the price breakdown reflects the override price in the total
    (e.g., if you selected 3 nights where one night = $250 and the others = base $120, total should be $250+$120+$120 = $490 nightly total, not $120×3=$360)
  </how-to-verify>
