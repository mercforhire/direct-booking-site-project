  <how-to-verify>
    Run `npm run dev` and test the following scenarios at http://localhost:3000:

    **Scenario 1 — Basic booking form navigation:**
    1. Visit any room listing (http://localhost:3000/rooms)
    2. Click a room → confirm "Request to Book" button is active (not disabled)
    3. Click it → confirm redirect to /rooms/[id]/book
    4. Confirm URL params (checkin, checkout, guests) are carried forward if set in the filter

    **Scenario 2 — Booking form layout and pricing:**
    1. On /rooms/[id]/book, confirm compact room summary at top (cover photo, room name, rate)
    2. Select a date range → confirm night count updates in price summary
    3. Change guest count → confirm extra guest fee row appears/disappears correctly
    4. Select an add-on checkbox → confirm add-on row appears in price summary
    5. Deselect add-on → confirm row disappears
    6. Confirm "Final price set by landlord at approval" footnote is visible
    7. Confirm desktop layout: form left + sticky price summary right (resize to mobile to confirm accordion)

    **Scenario 3 — Form validation:**
    1. Try to submit with no dates selected → confirm submit button is disabled
    2. Try to select a blocked date → confirm it's greyed out and not selectable
    3. Select a range shorter than minStayNights → confirm submit button is disabled

    **Scenario 4 — Booking submission (no account):**
    1. Select valid dates, fill all guest info fields (Name, Email, Phone)
    2. Leave "Create account" unchecked
    3. Submit → confirm redirect to /bookings/[id]?new=1
    4. Confirm success banner "Your request has been submitted"
    5. Confirm status badge shows "Pending"
    6. Confirm all booking details are shown (room name, dates, guests, add-ons, itemized costs)

    **Scenario 5 — Token-gated access:**
    1. Copy the /bookings/[id]?new=1 URL
    2. Open in a new incognito tab (no session) → with ?token= in the URL: confirm page loads
    3. Remove the ?token= param → confirm redirect to /guest/login

    **Scenario 6 — Booking submission (with account):**
    1. Submit a new booking, this time check "Create a free account" and enter a password
    2. Confirm booking creation succeeds
    3. Sign in at /guest/login with the email + password used during booking
    4. Visit /bookings/[id] (no token) → confirm page loads via session

    **Scenario 7 — Confirmation email (if RESEND_API_KEY is configured):**
    1. Check .env.local for RESEND_API_KEY and EMAIL_FROM
