# Plan 18-05: Guest Auth Route Migration

## Goal
Move all guest auth routes (`/guest/login`, `/guest/signup`, `/guest/forgot-password`, `/guest/reset-password`, `/auth/confirm`) under the `[landlord]` dynamic segment, update redirect logic to use `/{slug}/` prefixed paths, and update middleware to protect the new paths.

## must_haves
- [ ] Guest login page exists at `src/app/[landlord]/guest/login/page.tsx`
- [ ] Guest signup page exists at `src/app/[landlord]/guest/signup/page.tsx`
- [ ] Guest forgot-password page exists at `src/app/[landlord]/guest/forgot-password/page.tsx`
- [ ] Guest reset-password page exists at `src/app/[landlord]/guest/reset-password/page.tsx`
- [ ] Auth confirm route exists at `src/app/[landlord]/auth/confirm/route.ts`
- [ ] After login success, guest is redirected to `/{slug}/` (not hardcoded `/`)
- [ ] After logout from guest pages, redirect goes to `/{slug}/guest/login`
- [ ] Email confirmation redirect (`emailRedirectTo`) includes `/{slug}/auth/confirm`
- [ ] Old guest auth routes are deleted
- [ ] Middleware protects `/{slug}/my-bookings` and `/{slug}/bookings/` for logged-in guests

<task id="18-05-03">
<title>Move auth/confirm route to [landlord]/auth/confirm/ and keep root fallback</title>
<read_first>
- `src/app/auth/confirm/route.ts` — current email confirmation callback handler
</read_first>
<action>
1. Create `src/app/[landlord]/auth/confirm/route.ts`:

```typescript
import { type NextRequest, NextResponse } from "next/server"
import { createClient } from "@/lib/supabase/server"

export async function GET(
  request: NextRequest,
  { params }: { params: Promise<{ landlord: string }> }
) {
  const { landlord: slug } = await params
  const { searchParams } = new URL(request.url)
  const code = searchParams.get("code")
  const type = searchParams.get("type")
  const next = searchParams.get("next") ?? (type === "recovery" ? `/${slug}/guest/reset-password` : `/${slug}`)

  if (code) {
    const supabase = await createClient()
    const { error } = await supabase.auth.exchangeCodeForSession(code)
    if (!error) {
      return NextResponse.redirect(new URL(next, request.url))
    }
  }

  return NextResponse.redirect(new URL(`/${slug}/guest/login?error=invalid_token`, request.url))
}
```

2. Update the root `src/app/auth/confirm/route.ts` to be a fallback for admin OTP and legacy links:

```typescript
import { type NextRequest, NextResponse } from "next/server"
import { createClient } from "@/lib/supabase/server"

export async function GET(request: NextRequest) {
  const { searchParams } = new URL(request.url)
  const code = searchParams.get("code")
  const type = searchParams.get("type")
  const next = searchParams.get("next") ?? "/dashboard"

  if (code) {
    const supabase = await createClient()
    const { error } = await supabase.auth.exchangeCodeForSession(code)
    if (!error) {
      return NextResponse.redirect(new URL(next, request.url))
    }
  }

  return NextResponse.redirect(new URL("/login?error=invalid_token", request.url))
}
```

This keeps the root `/auth/confirm` for admin login flows (which redirect to `/dashboard`) and as a fallback for any existing email links. New guest auth flows will use `/{slug}/auth/confirm`.
</action>
