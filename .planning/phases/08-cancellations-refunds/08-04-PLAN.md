Payment difference logic on approval:
- Fetch booking.stripeSessionId + booking.confirmedPrice (amount already paid)
- newPrice > confirmedPrice: top-up needed
  - Stripe: call createDateChangeStripeCheckoutSession (returns { checkoutUrl }) — admin UI shows "Payment link sent to guest"
  - E-transfer: record topUpRequired on BookingDateChange (use newPrice field), update dates immediately
- newPrice < confirmedPrice: refund needed
  - Stripe: issue partial refund for (confirmedPrice - newPrice) using same refund pattern as cancelBooking
  - E-transfer: record refundRequired, update dates
- newPrice == confirmedPrice: update dates, done

Note: Per CONTEXT.md, "date change requests scope: any APPROVED or PAID booking". For APPROVED bookings (no payment taken yet), confirmedPrice may be null → treat as confirmedPrice = 0 for comparison.

Webhook for date_change_topup:
- Find BookingDateChange by dateChangeId from metadata
- Idempotent: skip if already status !== PENDING/APPROVED
- prisma.$transaction([bookingDateChange.update PAID, booking.update {checkin, checkout, confirmedPrice: newPrice}])

For approveDateChange, the price comparison logic:
```typescript
const priceDiff = newPrice - Number(dateChange.booking.confirmedPrice ?? 0)
const isStripe = !!dateChange.booking.stripeSessionId

if (priceDiff > 0) {
  if (isStripe) {
    // Create top-up Stripe session (do NOT update dates — webhook does it)
    const session = await stripe.checkout.sessions.create({
      mode: "payment",
      payment_method_types: ["card"],
      currency: "cad",
      line_items: [{ price_data: { currency: "cad", unit_amount: Math.round(priceDiff * 100), product_data: { name: `Date change top-up — ${booking.room.name}` } }, quantity: 1 }],
      metadata: { type: "date_change_topup", dateChangeId },
      success_url: `${origin}/bookings/${booking.id}?date_change_paid=1`,
      cancel_url: `${origin}/bookings/${booking.id}`,
      customer_email: booking.guestEmail,
    })
    await prisma.bookingDateChange.update({ where: { id: dateChangeId }, data: { stripeSessionId: session.id } })
    // Update booking dates and newPrice immediately even for Stripe top-up
    // so dates are reflected; payment confirms the full flow
    // Actually per research: "webhook updates booking.checkin + booking.checkout atomically"
    // So for Stripe top-up: do NOT update dates here, store checkoutUrl, return it
    return { success: true, checkoutUrl: session.url }
  } else {
    // E-transfer: update dates immediately
    await prisma.booking.update({ where: { id: booking.id }, data: { checkin: new Date(dateChange.requestedCheckin), checkout: new Date(dateChange.requestedCheckout), confirmedPrice: newPrice } })
  }
} else if (priceDiff < 0) {
  if (isStripe) {
    // Issue partial refund
    const session = await stripe.checkout.sessions.retrieve(booking.stripeSessionId!)
    if (session.payment_intent && typeof session.payment_intent === "string") {
      await stripe.refunds.create({ payment_intent: session.payment_intent, amount: Math.round(Math.abs(priceDiff) * 100) })
    }
  }
  // Update dates for both Stripe and e-transfer
  await prisma.booking.update({ where: { id: booking.id }, data: { checkin: new Date(dateChange.requestedCheckin), checkout: new Date(dateChange.requestedCheckout), confirmedPrice: newPrice } })
} else {
  // No price change: update dates
  await prisma.booking.update({ where: { id: booking.id }, data: { checkin: new Date(dateChange.requestedCheckin), checkout: new Date(dateChange.requestedCheckout) } })
}
```
