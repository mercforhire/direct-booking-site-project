Start dev server: `npm run dev`

**Scenario 1 — Cancel APPROVED booking from list:**
1. Open `/admin/bookings` in browser
2. Find a booking with status APPROVED
3. Click Cancel button in the row
4. Verify AlertDialog: "Cancel this booking?" — no refund amount field
5. Confirm → booking status changes to CANCELLED
6. Check guest booking page `/bookings/[id]?token=[accessToken]` — should show "This booking was cancelled. No payment was taken."

**Scenario 2 — Cancel APPROVED booking from detail page:**
1. Open `/admin/bookings/[id]` for an APPROVED booking
2. Find the Cancel section at bottom
3. Confirm simple dialog — no refund amount field for APPROVED bookings
4. Booking cancels successfully

**Scenario 3 — Cancel PAID+Stripe booking:**
1. Use a PAID booking with stripeSessionId set
2. On list page: PAID rows should show "Cancel (see refund)" link to detail page (NOT inline cancel)
3. On detail page: Cancel dialog should show refund amount input pre-filled with confirmedPrice
4. Verify deposit note shows correct text (pre-check-in or mid-stay based on today vs checkin)
5. Verify Stripe timeline note: "Stripe refunds typically take 5–10 business days."
6. Submit with the pre-filled amount — confirm Stripe refund issued in Stripe Dashboard

**Scenario 4 — Stripe refund failure:**
1. Use a test booking with an invalid stripeSessionId (or simulate by temporarily breaking the Stripe key)
2. Attempt cancellation via detail page
3. Verify: error message shown in dialog, booking still shows PAID (NOT cancelled)

**Scenario 5 — Cancel PAID+e-transfer booking:**
1. Use a PAID booking with stripeSessionId null (e-transfer payment)
2. Cancel dialog: enter refund amount
3. Confirm — no Stripe call; booking moves to CANCELLED
4. Guest page: "Refund of $X will be sent via e-transfer."

**Scenario 6 — Guest date change request:**
1. Go to a guest booking page for an APPROVED or PAID booking
2. Find the "Request Date Change" section
3. Enter new check-in / checkout dates and optional note
4. Submit → status shows "Date change requested: [dates] — Awaiting landlord approval"
5. Cancel the request → section returns to "Request Date Change" form

**Scenario 7 — Landlord approve/decline date change:**
1. After guest submits, open `/admin/bookings/[id]`
2. Find the Date Change Request section
3. Test Approve: enter new price → confirm
4. Test Decline: enter optional reason → confirm
5. Guest page reflects updated status
