## Files Created/Modified

- `src/middleware.ts` - Supabase SSR updateSession pattern; protects /dashboard, /rooms, /settings using getUser(); replaced NextAuth edgeAuth import
- `src/app/auth/confirm/route.ts` - GET handler for magic link callback; calls verifyOtp with token_hash and type; redirects to /dashboard or /login?error=invalid_token

## Decisions Made
- Middleware creates its own inline createServerClient rather than importing from src/lib/supabase/server — this is intentional because the middleware must update cookies on both the request object (for Server Components) and the response object (for the browser), which requires direct access to both that the shared factory cannot provide
- getUser() chosen over getSession() for middleware protection — getUser() makes a server-side JWT validation call to Supabase Auth, while getSession() only reads the local cookie value and could be spoofed
