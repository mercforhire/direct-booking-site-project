<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create Supabase project and configure credentials</name>
  <what-to-do>
    Complete these steps in order before continuing:

    1. Go to supabase.com and create a new project. Choose a region closest to your deployment target (us-east-1 / AWS US East, or ca-central-1 / AWS Canada). Note the database password.

    2. In Supabase Dashboard > Authentication > Users > click "Add User" (Invite user). Enter the landlord email address. This is the only account that will receive magic links.

    3. In Supabase Dashboard > Authentication > SMTP Settings, enable custom SMTP and enter:
       - Host: smtp.resend.com
       - Port: 465
       - Username: resend
       - Password: your RESEND_API_KEY value (from existing .env.local)
       - Sender email: your existing AUTH_FROM_EMAIL value
       Save and confirm.

    4. In Supabase Dashboard > Authentication > URL Configuration > Redirect URLs, add:
       - http://localhost:3000/**
       - Your production URL/** (if known)

    5. Go to Project Settings > API. Copy these values into .env.local:
       - Project URL → NEXT_PUBLIC_SUPABASE_URL
       - anon public key → NEXT_PUBLIC_SUPABASE_ANON_KEY
       - service_role key → SUPABASE_SERVICE_ROLE_KEY

    6. Go to Project Settings > Database > Connection string. Copy:
       - "Transaction" tab (port 6543) — add ?pgbouncer=true to the end → DATABASE_URL
       - "Session" tab (port 5432) — no extra params → DIRECT_URL

    7. In .env.local, REMOVE these lines (they are NextAuth vars that will no longer be used):
       - NEXTAUTH_SECRET
       - NEXTAUTH_URL
       - AUTH_RESEND_KEY
       - AUTH_FROM_EMAIL

    8. Keep RESEND_API_KEY (still needed for Supabase SMTP) and all UPLOADTHING_* vars.

    Final .env.local should have: NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY, DATABASE_URL (port 6543 + ?pgbouncer=true), DIRECT_URL (port 5432), RESEND_API_KEY, UPLOADTHING_SECRET, UPLOADTHING_APP_ID.
  </what-to-do>
  <resume-signal>Type "done" when .env.local is updated with all Supabase credentials and the landlord user is created in Supabase Auth.</resume-signal>
</task>
