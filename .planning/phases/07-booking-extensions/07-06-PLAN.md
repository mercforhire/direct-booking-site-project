Component logic:

**Show "Request Extension" form when:**
- `booking.status` is "APPROVED" or "PAID"
- AND `activeExtension` is null OR `activeExtension.status === "DECLINED"`

**Form state:** `showForm` boolean toggled by "Request Extension" button. Initially false.

**Form fields:** DayPicker (single mode) + optional textarea for noteToLandlord.

**DayPicker disabled prop:** `[{ before: new Date(booking.checkout) }, ...blockedDates]`

Note on DayPicker CSS import: Use `import "react-day-picker/style.css"` — this is the react-day-picker v9 import path. If this causes a server-component error, move the import to a dedicated CSS file or use the existing global stylesheet import pattern in the project.

**After submit:** Set `submitted = true` state — renders success message "Extension request submitted — we'll email you once it's reviewed." instead of form. Do NOT use router.push or redirect.

**PENDING state:** Show "Extension requested to [date] — awaiting review" and Cancel button (wrapped in AlertDialog for confirmation).

**APPROVED state:** Show extension price + Stripe Pay button (calls `createExtensionStripeCheckoutSession(extension.id)` via useTransition) + e-transfer instructions with extensionPrice, etransferEmail, extension.id as reference.

**DECLINED state:** Show "Extension request declined" badge + declineReason if provided. Re-enable the "Request Extension" button.

**PAID state:** Show "Extension paid" message with the new checkout date from `extension.requestedCheckout`.

Section header: "Extension Request" — rendered as `<h2>` in a styled container matching the PaymentSection style (mt-6 space-y-4 pattern from booking-status-view.tsx).
